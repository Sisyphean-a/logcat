# H5 / Android Logcat Viewer 开发文档

## 1. 项目目标

开发一个桌面端日志查看工具，用于在电脑上实时查看 Android 设备中的 Logcat 日志。

工具目标不是简单封装 `adb logcat`，而是实现接近 Android Studio Logcat 的完整体验，并额外强化 H5 / WebView 调试场景。

核心目标：

1. 支持 USB / Wi-Fi 连接 Android 设备。
2. 支持实时查看 Logcat 日志。
3. 支持按设备、App、进程、tag、level、message、时间范围过滤。
4. 支持类似 Android Studio Logcat 的查询语法。
5. 支持日志高亮、搜索、暂停、清空、自动滚动、复制、导出。
6. 支持多标签、多窗口、多设备、多查询同时查看。
7. 支持 Android WebView / H5 场景，能够更方便地过滤 `chromium` 日志。
8. 支持保存过滤器、历史查询、常用工作区。
9. 支持崩溃日志、异常堆栈、多行日志合并展示。
10. 支持轻量运行，不依赖 Android Studio。

------

## 2. 推荐技术方案

### 2.1 首选方案

推荐使用：

```text
Electron + Node.js + TypeScript + React / Vue
```

原因：

1. 前端开发友好。
2. 能直接调用本机 `adb` 命令。
3. 能做完整桌面 GUI。
4. 跨平台支持 Windows / macOS / Linux。
5. 后续可以打包成独立安装包。
6. UI 能做到接近 Android Studio Logcat。

### 2.2 可选方案

#### VSCode 扩展

适合 H5 开发者，优点是直接嵌入 VSCode。

缺点：

1. UI 自由度不如 Electron。
2. 多窗口、多设备、多标签能力受 VSCode 面板限制。
3. 不适合做成完全媲美 Android Studio 的独立工具。

#### Tauri

更轻量，体积小。

缺点：

1. Rust 后端门槛比 Node 高。
2. 对你当前前端开发习惯来说，Electron 更快落地。

#### 纯命令行 TUI

比如 Node + Ink / Go + Bubble Tea。

缺点：

1. 不适合复杂筛选。
2. 不适合媲美 Android Studio Logcat。
3. 不适合图形化多窗口、多标签、多条件过滤。

最终建议：

```text
如果目标是完整工具：Electron
如果目标是自己用的轻量工具：VSCode 扩展
如果目标是先做 MVP：Node CLI + 简单 Web UI
```

## 4. 核心使用场景

### 4.1 H5 开发者查看 WebView 日志

用户在 H5 代码中：

```ts
console.log('[H5]', '进入申请页')
console.error('[H5]', '接口失败', error)
```

工具中选择：

```text
设备：SM-A217F
App：宿主 App 包名
Tag：chromium
Message：包含 [H5]
```

最终只看到 H5 日志。

------

### 4.2 查看某个 App 的全部日志

用户选择某个包名，例如：

```text
com.xxx.app
```

工具自动获取该 App 当前 PID，并执行类似：

```bash
adb logcat --pid=<pid>
```

App 重启后，工具自动重新识别 PID 并继续监听。

------

### 4.3 查看崩溃日志

用户选择：

```text
level >= ERROR
buffer = crash
```

工具自动展示异常堆栈，支持折叠/展开。

------

### 4.4 多设备同时查看

同时连接两台手机：

```text
Pixel 7a
Samsung SM-A217F
```

工具支持为每台设备打开独立日志标签。

------

### 4.5 多查询并行

用户可以同时打开：

```text
Tab 1：H5 日志
Tab 2：网络错误
Tab 3：Android Bridge 日志
Tab 4：崩溃日志
```

每个 Tab 有独立的设备、App、过滤条件、展示配置。

## 6. ADB 能力设计

### 6.1 ADB 检测

启动时检测：

1. 系统 PATH 中是否存在 `adb`。
2. 用户配置的 adb 路径是否有效。
3. Android SDK 是否存在。
4. 当前 adb 版本。
5. adb server 是否运行。

检测命令：

```bash
adb version
adb devices -l
```

如果 adb 不存在，提示用户：

```text
未检测到 adb，请安装 Android Platform Tools，或手动配置 adb 路径。
```

------

### 6.2 设备列表

执行：

```bash
adb devices -l
```

解析结果：

```text
设备 ID
设备状态
设备型号
连接方式
USB / Wi-Fi
```

设备状态包括：

```text
device
offline
unauthorized
no permissions
```

UI 展示：

```text
SM-A217F    device       usb
Pixel 7a    unauthorized usb
192.168.0.101:5555 device wifi
```

------

### 6.3 设备授权提示

如果设备状态是：

```text
unauthorized
```

提示：

```text
手机未授权，请查看手机屏幕，允许 USB 调试。
```

如果是：

```text
offline
```

提示：

```text
设备离线，请重新插拔 USB，或执行 adb kill-server / adb start-server。
```

------

### 6.4 Wi-Fi ADB 支持

支持手动连接：

```bash
adb connect 192.168.0.101:5555
```

UI 功能：

```text
连接 Wi-Fi 设备
断开 Wi-Fi 设备
保存历史 IP
自动重连
```

## 7. App / 包名 / 进程管理

### 7.1 获取已安装 App

执行：

```bash
adb shell pm list packages
```

支持过滤：

```text
用户 App
系统 App
全部 App
```

常用命令：

```bash
adb shell pm list packages -3
adb shell pm list packages -s
adb shell pm list packages
```

------

### 7.2 获取 App 名称

可选实现：

```bash
adb shell dumpsys package <packageName>
```

或者后续通过 APK 信息解析。

MVP 阶段可以只显示包名。

------

### 7.3 获取当前前台 App

执行：

```bash
adb shell dumpsys window | findstr mCurrentFocus
```

或者：

```bash
adb shell dumpsys activity activities
```

功能：

```text
一键选择当前前台 App
```

这个对 H5 调试非常有用，因为用户可能不知道宿主 App 包名。

------

### 7.4 获取进程 PID

执行：

```bash
adb shell pidof -s <packageName>
```

如果多进程：

```bash
adb shell pidof <packageName>
```

需要支持：

```text
主进程
WebView 进程
子进程
远程 service 进程
```

------

### 7.5 App 重启监听

当 App 崩溃或重启后，PID 会变化。

工具需要自动检测：

```text
PROCESS ENDED
PROCESS STARTED
PID changed
```

然后自动重新绑定新 PID。

UI 显示：

```text
App 已重启，已切换到新进程 12345
```

## 8. Logcat 采集设计

### 8.1 基础命令

默认使用：

```bash
adb -s <deviceId> logcat -v threadtime
```

推荐格式：

```bash
adb -s <deviceId> logcat -v threadtime
```

原因：

```text
包含日期、时间、PID、TID、level、tag、message
```

------

### 8.2 按 tag 过滤

例如只看 WebView：

```bash
adb -s <deviceId> logcat -v threadtime -s chromium:I "*:S"
```

------

### 8.3 按 PID 过滤

例如只看某 App：

```bash
adb -s <deviceId> logcat -v threadtime --pid=<pid>
```

注意：

```text
--pid 依赖设备系统版本。
部分旧设备可能不支持。
```

如果不支持，需要降级方案：

1. 全量读取 logcat。
2. 本地解析 PID。
3. 在工具内过滤 PID。

------

### 8.4 按 buffer 读取

支持 buffer：

```text
main
system
crash
radio
events
all
default
```

命令示例：

```bash
adb logcat -b main
adb logcat -b crash
adb logcat -b main,system,crash
```

UI 支持多选 buffer。

------

### 8.5 清空日志

执行：

```bash
adb -s <deviceId> logcat -c
```

UI 功能：

```text
清空当前设备日志
清空当前视图
```

注意区分：

```text
清空设备日志：真正清掉设备 log buffer
清空当前视图：只清掉工具界面里的显示内容
```

------

### 8.6 暂停与恢复

暂停不应该杀掉 adb 进程。

推荐做法：

```text
采集继续进行
UI 暂停追加
后台缓存最近 N 条
恢复后补充显示
```

配置项：

```text
暂停时最大缓存条数：10000
超过后丢弃旧日志
```

------

### 8.7 自动滚动

支持：

```text
自动滚动到底部
用户手动滚动后自动暂停跟随
点击按钮恢复跟随
```

类似 Android Studio 行为。

## 9. 日志解析设计

### 9.1 threadtime 格式

典型日志：

```text
06-04 16:42:18.479 10665 10665 I chromium: [INFO:CONSOLE(618)] "[vite] connected.", source: http://192.168.0.196:5173/@vite/client (618)
```

解析字段：

```ts
interface LogEntry {
  id: string
  deviceId: string
  timestamp: string
  pid: number
  tid: number
  level: 'V' | 'D' | 'I' | 'W' | 'E' | 'F'
  tag: string
  packageName?: string
  processName?: string
  message: string
  raw: string
  buffer?: string
}
```

------

### 9.2 WebView console 日志解析

WebView console 常见格式：

```text
I chromium: [INFO:CONSOLE(123)] "hello", source: http://xxx/index.js (123)
```

需要额外解析：

```ts
interface WebViewConsoleInfo {
  consoleLevel: 'INFO' | 'WARNING' | 'ERROR'
  lineNumber?: number
  sourceUrl?: string
  sourceLine?: number
  consoleMessage: string
}
```

展示时可以优化成：

```text
[H5] hello
source: index.js:123
```

------

### 9.3 多行日志合并

异常堆栈通常是多行：

```text
E AndroidRuntime: FATAL EXCEPTION: main
E AndroidRuntime: java.lang.RuntimeException: xxx
E AndroidRuntime:     at com.xxx.MainActivity.onCreate(MainActivity.java:20)
```

需要合并为一个可折叠块。

规则：

1. 如果下一行是堆栈缩进，合并到上一条。
2. 如果 tag / pid 相同且时间连续，可以合并。
3. 支持展开 / 折叠。
4. 支持复制完整堆栈。

------

### 9.4 JSON 日志识别

H5 经常打印 JSON：

```text
[H5] 接口返回 {"code":0,"data":{"name":"test"}}
```

工具应支持：

1. 自动识别 JSON。
2. 右侧详情面板格式化 JSON。
3. 支持折叠字段。
4. 支持复制格式化 JSON。
5. 支持复制原始 JSON。

------

### 9.5 URL 识别

日志中出现 URL：

```text
http://192.168.0.196:5173/apply
https://api.xxx.com/order/list
```

支持：

1. 自动高亮 URL。
2. 点击复制。
3. 可选：用浏览器打开。
4. 可选：解析 query 参数。



```text

```

看接口相关：

```text
message:api | message:request | message:response
```

看最近 5 分钟：

```text
age:5m
```

## 34. 关键实现伪代码

### 34.1 启动 Logcat

```ts
import { spawn } from 'child_process'

function startLogcat(deviceId: string) {
  const child = spawn('adb', [
    '-s',
    deviceId,
    'logcat',
    '-v',
    'threadtime'
  ])

  child.stdout.on('data', chunk => {
    handleLogcatChunk(chunk)
  })

  child.stderr.on('data', chunk => {
    handleLogcatError(chunk)
  })

  child.on('exit', code => {
    handleLogcatExit(code)
  })

  return child
}
```

------

### 34.2 按 chromium 过滤

```ts
function startChromiumLogcat(deviceId: string) {
  return spawn('adb', [
    '-s',
    deviceId,
    'logcat',
    '-v',
    'threadtime',
    '-s',
    'chromium:I',
    '*:S'
  ])
}
```

------

### 34.3 解析日志行

```ts
function parseThreadtimeLine(line: string): LogEntry | null {
  const reg = /^(\d{2}-\d{2})\s+(\d{2}:\d{2}:\d{2}\.\d{3})\s+(\d+)\s+(\d+)\s+([VDIWEF])\s+([^:]+):\s?(.*)$/

  const match = line.match(reg)
  if (!match) return null

  return {
    id: crypto.randomUUID(),
    deviceId: '',
    timestamp: Date.now(),
    timeText: `${match[1]} ${match[2]}`,
    pid: Number(match[3]),
    tid: Number(match[4]),
    level: match[5] as LogEntry['level'],
    tag: match[6].trim(),
    message: match[7],
    raw: line
  }
}
```

------

### 34.4 H5 过滤

```ts
function isH5Log(entry: LogEntry) {
  return entry.tag === 'chromium' && entry.message.includes('[H5]')
}
```

------

### 34.5 WebView Console 解析

```ts
function parseWebViewConsole(message: string): WebViewConsoleInfo | null {
  const reg = /\[(INFO|WARNING|ERROR):CONSOLE\((\d+)\)\]\s+"(.*)",\s+source:\s+(.*)\s+\((\d+)\)/

  const match = message.match(reg)
  if (!match) return null

  return {
    consoleLevel: match[1] as WebViewConsoleInfo['consoleLevel'],
    lineNumber: Number(match[2]),
    consoleMessage: match[3],
    sourceUrl: match[4],
    sourceLine: Number(match[5])
  }
}
```

## 38. 最终目标验收标准

3. 能按 level、tag、package、process、message、age 查询。
4. 支持正则、否定、AND、OR、括号。
5. 支持 pause、restart、clear、scroll to end。
6. 支持 compact view、字段显示配置、颜色高亮。
7. 支持多 Tab、多查询、分屏。
8. 支持崩溃堆栈识别。
9. 支持日志导出。
10. 支持设备断开重连。
11. 支持 App 重启后继续跟踪。
12. 支持 WebView / H5 日志优化。
13. 支持大日志量不卡顿。
14. 支持保存过滤器和工作区。
15. 支持 Windows/macOS/Linux 打包。
